# Final evaluation report

Snapshot of the EN→ES blast-equipment translator after the **v4** sweep —
ALL-CAPS preprocessing fix on the production pipeline (Marian-CT2 + Qwen
2.5-3B-Instruct post-edit), and the side-by-side comparison against
NLLB-200-distilled-600M and NLLB-200-distilled-1.3B that motivated keeping
Marian as the default.

Regenerated by `scripts/run_full_eval.py` (writes `results_final/`).

For the broader pipeline narrative — including the four iteration steps
that produced these numbers and the remaining failure classes — see the
project README (`README_ENG.md` § *Detailed analytics*).

## Engine sweep — why Marian beats NLLB on this corpus

A three-way evaluation was run on the same 1,334-segment corpus, with
identical glossary + ALL-CAPS preprocessing + Qwen post-edit. Only the MT
backend changes.

| Engine | LaBSE | BERTScore | COMET w20† | Broken† | Speed | Score* |
|---|---:|---:|---:|---:|---:|---:|
| **Marian opus-mt-en-es (CT2)** | **0.9007** | **0.9111** | −0.005 | **3 (0.2 %)** | **0.65 seg/s** | **0.8234** |
| NLLB-200-distilled-600M (HF) | 0.8962 | 0.9106 | +0.008 | 12 (0.9 %) | 0.55 seg/s | 0.8217 |
| NLLB-200-distilled-1.3B (HF) | 0.8920 | 0.9089 | **+0.020** | 8 (0.6 %) | 0.50 seg/s | 0.8197 |

† **wmt20** COMET-QE **z-score** (same calibration as `run_comet_on_worst.py`). The **corpus headline** uses **wmt22 Kiwi** (~0–1); see the metrics table at the top.

\* *Score = 0.5·LaBSE + 0.3·BERTScore + 0.2·σ(COMET-QE). All three are in
the [0,1] range so the convex combination is well-defined.*

Interpretation:

* **NLLB is more fluent on average** (higher **wmt20** COMET-QE z-score —
  the output reads more like idiomatic news Spanish), but it **drifts further from
  the source semantics** (lower LaBSE / BERTScore) and has **2–4× more
  catastrophic failures** (broken rows on that z-score scale). For technical safety
  documentation, where literal fidelity matters more than fluency, Marian
  wins.
* **NLLB-1.3B does *not* dominate NLLB-600M** — the larger model paraphrases
  more aggressively, which hurts semantic similarity to the (often terse)
  source.
* **Glossary hit rate** is roughly equal across engines (mid‑90s %); the
  glossary plumbing is engine-agnostic so this is expected.
* **Speed**: Marian-CT2 is 1.3× faster than NLLB-1.3B at identical batch
  size, with no quality penalty.

Decision: **keep Marian opus-mt-en-es (CT2) as the default**. NLLB-200 is
still selectable via `MT_MVP_MT_ENGINE=nllb` for users who prefer fluency
over fidelity; see `README_ENG.md` § *Configuration*.
